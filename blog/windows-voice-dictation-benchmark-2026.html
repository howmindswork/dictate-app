Breakdown: Where Latency Happens

The total ~200ms splits across five stages. Understanding this explains why some competitors are slower.

Hotkey Detection: ~1–2ms

Windows detects your Ctrl+Shift+Space press and wakes the dictate.app process. Native hotkey handlers have minimal latency.

Audio Capture: ~0–5ms

dictate.app captures microphone input via Windows Audio API. Recording stops when you release the hotkey. Buffering is near-instant.

API Transmission: ~20–50ms

WAV buffer travels from your PC to Groq's endpoint. Latency depends on your ISP, packet loss, and server proximity. Typical: 30ms.

Whisper Inference: ~150–180ms

Groq's optimized Whisper model runs on specialized silicon. This is where most latency lives. OpenAI Whisper API averages 350–800ms here because they route through shared cloud infrastructure with higher contention.

Text Injection: ~10–20ms

Groq returns transcribed text. dictate.app injects it at your cursor via Windows SendKeys API. This is nearly instant.

Why Groq Hardware Is Different

Groq doesn't use standard GPU infrastructure. Their LPU (Language Processing Unit) is application-specific silicon designed for LLM inference.

Result: Whisper runs at peak efficiency. No queuing. No contention from other workloads. No resource sharing. ~200ms is consistent.

OpenAI Whisper API runs on shared cloud GPUs. Your request waits in queue with others. Inference time varies 400–900ms depending on server load.

Local Whisper: Why It's Slower Than You'd Think

You'd expect local Whisper to be fastest. It's not. Even on a gaming PC with RTX 4060 GPU:

  • Model loading: First inference loads model from disk. 300–600ms overhead.
  • GPU warm-up: Batch processing optimizations kick in after 5–10 inferences.
  • CPU inference fallback: If GPU is in use, Whisper falls back to CPU. 2–4 second latency.
  • Audio buffering: Capturing and converting WAV adds 20–50ms.

Net: Local Whisper 800–1,200ms. Faster than API calls, slower than Groq.

The Competitor Transparency Gap

DictaFlow: Latency not published. This is telling. If they were faster than 400ms, they'd advertise it. They route through their own servers, adding processing overhead.

Dragon NaturallySpeaking: Desktop software. Decent accuracy, but latency is variable and depends on system load. Typical 100–300ms, but often slower on older machines.

Windows Native (Win+H): Uses cloud speech-to-text. No latency data published. Requires manual copy-paste from clipboard, adding 2–5 seconds of UX friction.

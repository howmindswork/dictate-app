How dictate.app Achieves This Speed
  1. Direct API: Your audio goes straight to Groq's Whisper API. No intermediary, no queuing.
  2. Groq's Hardware: Groq optimizes LLM inference on specialized silicon. ~200ms is their standard Whisper latency.
  3. BYOK Architecture: You bring your own API key. We never process, store, or see your audio. Just metadata.
  4. Windows Native: Push-to-talk stays in memory as WAV. Transcription result types at cursor instantly. No clipboard, no app switching.

This is why no other Windows dictation app can claim ~200ms latency without routing through their own servers (which adds delay) or using the same Groq direct API (which they don't).

Measurement Methodology

Our benchmark measures the complete path from voice input to text output:

  1. Hotkey Detection: Windows receives your hotkey press (Global hotkey handler, ~1-2ms overhead)
  2. Audio Capture: dictatе.app captures microphone input via Windows Audio API, stopping at hotkey release (~0-5ms)
  3. API Transmission: WAV buffer sent to Groq Whisper endpoint over your Internet connection (~20-50ms depending on ISP)
  4. Whisper Transcription: Groq processes audio on specialized LLM inference hardware (~150-180ms)
  5. Response & Text Injection: Groq returns transcribed text, inject.js types it at the cursor position (~10-20ms)

Measurements taken May 2026 with Groq Whisper API on Windows 11 via dictate.app. Competitor claims sourced from public documentation and vendor statements.
